# Data Dictionary — Container Dwell Time Analytics Skill

## Data Source

This skill uses **synthetic container movement data** generated by
`scripts/generate_synthetic_data.py`. The generator creates realistic terminal
operations data modelling:

- Realistic container size distributions (~60% 40ft, ~40% 20ft)
- Seasonal gate volume patterns (weekly cycles, monthly peaks)
- Movement type distributions typical of a mid-size multi-purpose terminal
- Known ground-truth dwell clusters for validation (planted overstay groups)
- Configurable terminal layout, date range, and container volume

The synthetic data is designed to exercise all pipeline stages including edge cases
(long-stay empties, reefer plug saturation, vessel surge events).

For real-world use, the same CSV schema accepts data exported from any Terminal
Operating System (TOS).

## Container Movement Records — Column Definitions

### Required Columns

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `container_id` | string | Not null; BIC format preferred | Unique container identifier (e.g., MSCU7234561) |
| `size_ft` | integer | Values: 20 or 40 only | Physical container length in feet |
| `container_type` | string | Values: dry, reefer, hazardous, empty | Cargo/container classification |
| `weight_tons` | float | Range: 1.0–45.0; not null | Gross weight in metric tons |
| `gate_in_time` | datetime | ISO 8601 format; not null | Timestamp when container entered the terminal |
| `gate_out_time` | datetime | ISO 8601 format; null if still in yard | Timestamp when container left the terminal |
| `movement_type` | string | Values: import, export, transhipment | Direction of cargo flow |
| `yard_block` | string | Not null | Block identifier where container was stored |

### Optional Columns

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `yard_bay` | integer | Positive integer | Bay position within block |
| `yard_row` | integer | Positive integer | Row position within block |
| `yard_tier` | integer | Range: 1–6 typical | Stack height position (1 = ground) |
| `vessel_name` | string | Free text | Name of associated vessel |
| `shipping_line` | string | Free text | Carrier / shipping line name |
| `cargo_category` | string | Free text | Cargo type (food, chemicals, electronics, etc.) |
| `gate_in_mode` | string | Values: vessel, truck, rail, barge | How the container arrived at terminal |
| `gate_out_mode` | string | Values: vessel, truck, rail, barge | How the container departed terminal |

## Yard Configuration — JSON Schema

```json
{
  "terminal_name": "string — terminal identifier for report headers",
  "blocks": [
    {
      "block_id": "string — must match yard_block values in CSV",
      "rows": "int — number of rows in the block",
      "bays": "int — number of bays in the block",
      "max_tier": "int — maximum stacking height allowed",
      "teu_capacity": "int — total TEU slots available",
      "accepts_types": ["list of container_type values this block accepts"],
      "has_reefer_plugs": "boolean — whether block has power connections",
      "reefer_plug_count": "int — number of reefer power plugs (0 if no plugs)"
    }
  ],
  "gate_count_in": "int — number of inbound gate lanes",
  "gate_count_out": "int — number of outbound gate lanes"
}
```

## Tariff Configuration — JSON Schema

```json
{
  "currency": "string — ISO 4217 code (e.g., USD, EUR, INR)",
  "free_days": {
    "import": "int — free storage days for import containers",
    "export": "int — free storage days for export containers",
    "transhipment": "int — free storage days for transhipment containers"
  },
  "storage_tiers": [
    {
      "from_day": "int — start of tier (inclusive, after free days)",
      "to_day": "int or null — end of tier (inclusive, null = unlimited)",
      "rate_per_teu_per_day": "float — charge per TEU per day in this tier"
    }
  ],
  "reefer_surcharge_per_day": "float — additional daily charge per reefer TEU",
  "hazardous_surcharge_per_day": "float — additional daily charge per hazmat TEU",
  "avg_throughput_revenue_per_teu": "float — average handling revenue per TEU moved"
}
```
